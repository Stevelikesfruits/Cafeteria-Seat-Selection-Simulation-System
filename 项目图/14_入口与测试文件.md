## main_ui.py -- 程序入口

```mermaid
classDiagram
    class main_ui_py {
        <<启动脚本>>
        --
        sys.path 初始化
        QApplication 创建
        MainWindow 实例化
        app.exec() 事件循环
    }
    main_ui_py --> MainWindow : 创建并显示
```

### 启动流程

```mermaid
flowchart TD
    A["1. sys.path.append()<br/>将项目根目录加入导入路径"] --> B["2. QApplication(sys.argv)<br/>创建 Qt 应用程序"]
    B --> C["3. MainWindow()<br/>创建主窗口实例"]
    C --> D["4. window.show()<br/>显示窗口"]
    D --> E["5. app.exec()<br/>进入 Qt 事件循环<br/>阻塞直到窗口关闭"]
    E --> F["sys.exit() 安全退出"]
```

核心就 6 行代码，典型的 PySide6 最小启动模板。

## main_test.py -- 后台脱机测试

```mermaid
classDiagram
    class main_test_py {
        <<测试脚本>>
        --
        run_backend_test()
        SimulationEngine 直接使用
        for 60 分钟循环
        每5分钟打印快照
    }
    main_test_py --> SimulationEngine : 直接调用 step() *60
```

### 测试流程

```mermaid
flowchart TD
    A["开始测试"] --> B["创建 SimulationEngine 默认配置"]
    B --> C["仿真时长: 60 分钟"]
    C --> D["循环 t=1 -> 60"]
    D --> E["engine.step()<br/>推进1分钟"]
    E --> F{"t%5==0 or t==1?"}
    F -->|"是"| G["打印快照:<br/>当前就餐人数<br/>累计处理人数<br/>满意度得分"]
    F -->|"否"| H["继续"]
    G --> H
    H --> D
    D -->|"循环结束"| I["输出最终报告<br/>总时间/总人数/满意度"]
```

无 GUI 依赖，纯控制台输出，用于验证后端逻辑是否正常。

```

**关键观察**：`main_test.py` 和 `main_ui.py` 都依赖 `SimulationEngine`，但走不同路径：
- GUI 路线：`main_ui.py -> MainWindow -> QTimer -> SimulationEngine.step()`
- 测试路线：`main_test.py -> for 循环 -> SimulationEngine.step()` 直接调用
```
