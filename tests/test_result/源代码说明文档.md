# 餐厅排座仿真系统 — 源代码说明文档

## 一、项目概述

本项目模拟餐厅高峰期学生就餐排座场景。系统根据用户设定的偏好比例生成学生，按照座位偏好自动分配座位，计算满意度，并通过 GUI 实时展示餐厅座位状态。

**技术栈**: Python 3.9 / PySide6 / numpy / matplotlib / pytest

---

## 三、Models 层 — 数据模型

### 3.1 `student.py`

**PreferenceType** — 座位偏好枚举，4 个成员:

| 成员 | 含义 |
|------|------|
| `SINGLE` | 单人单座 |
| `FACE_TO_FACE` | 双人面对面 |
| `DIAGONAL` | 双人斜对角 |
| `ADJACENT` | 双人邻座 |

**Student** — 学生数据类 (`@dataclass`)，核心字段:

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | int | 唯一标识，全局自增 |
| `preference` | PreferenceType | 座位偏好 |
| `arrival_time` | int | 到达时间 (分钟) |
| `seated_table_id` | int | 落座桌子 ID，-1=未落座 |
| `seated_seat_indices` | List[int] | 占用的具体座位索引 |
| `leave_time` | int | 离开时间 (到达+20分钟) |
| `satisfaction_score` | int | 满意度得分 (+2/+1/-1) |

属性 `is_seated`: 判断 `seated_table_id != -1`。

### 3.2 `restaurant.py`

**Table** — 桌子基类。核心数据 `seats` 是 numpy 一维数组，0=空位，非0=占用者 student.id。提供 `get_free_seats()`、`occupied_count`、`is_full` 三个查询方法。

**Table2 / Table4** — 继承 Table，仅通过 `super().__init__` 传入不同参数:
- Table2: capacity=2, space_cost=4
- Table4: capacity=4, space_cost=6

**Restaurant** — 餐厅容器。维护 `tables: Dict[int, Table]` 字典，构造函数根据用户指定的双人桌/四人桌数量创建桌子，ID 从 1 递增。总空间上限 200 单位。

## 四、Core 层 — 业务逻辑

### 4.1 `simulation.py` — 仿真引擎

**SimulationEngine** 是系统的核心调度器，持有 Restaurant、StudentGenerator、SeatAllocator 三个子模块。

核心方法 `step()` 每调用一次推进 1 分钟，执行三个阶段:
1. 处理离店 — 遍历 active_students，到期者释放座位并移入 history
2. 生成新生 — 调用 Generator 按当前时间和偏好比例生成一批学生
3. 分配座位 — 逐个调用 Allocator，成功则设定 leave_time 和满意度分数，失败则直接归档

其他方法:
- `update_preferences(dict)` — 接收 UI 传入的新偏好比例
- `get_statistics()` — 返回当前时间、就餐人数、累计处理数、满意度总分
- `reset()` — 重建所有子模块，清空状态，恢复默认偏好

### 4.2 `student_generator.py` — 学生生成器

**StudentGenerator** 维护一个全局自增的 `current_student_id`。

`generate_batch(current_time, pref_ratios)`:
- 使用倒 U 型函数 `y = -0.05(t-30)^2 + 15` 计算基础人数，叠加随机扰动 [-2, +3]
- 按 `pref_ratios` 字典中的权重，使用 `random.choices` 为每个学生分配偏好
- 返回该批次的 Student 列表

高峰期在 t=30 分钟，约 15 人/分钟；低谷期人数自动降为 0。

### 4.3 `seat_allocator.py` — 座位分配器

**SeatAllocator** 持有 Restaurant 引用，采用两阶段策略:

`allocate(student)`:
1. **完美匹配** — `_find_perfect_match(pref)`：遍历所有桌子，按偏好类型匹配
   - SINGLE: 找完全空桌
   - FACE_TO_FACE: Table2 有任意空位即匹配；Table4 优先对面空位 (0↔2, 1↔3)
   - DIAGONAL / ADJACENT: Table4 有任意空位即匹配
2. **降级匹配** — `_find_any_free_seat()`：找第一个有空位的桌子，不检测偏好

匹配成功后写入 `table.seats[idx] = student.id` 并更新 Student 的落座信息。

### 4.4 `satisfaction.py` — 满意度计算器

**SatisfactionCalculator** 是纯静态类，无状态。

`calculate_and_assign(student, is_perfect_match)`:
- 未落座 → -1 分
- 落座且完美匹配 → +2 分
- 落座但降级匹配 → +1 分

直接修改 Student 对象的 `satisfaction_score` 字段。

## 五、UI 层 — 图形界面

### 5.1 `main_window.py` — 主窗口

**MainWindow** (QMainWindow 子类) 是 UI 层的总控制器。

启动流程: 弹出桌数设置弹窗 → 创建 SimulationEngine → 创建 QTimer(500ms) → 构建界面布局 → 初始化操控面板和餐厅视图。

布局: 顶部按钮栏 (开始/暂停/结束) + 左侧餐厅视图 + 右侧操控面板。

核心交互:
- **开始**: 校验偏好总和=100%，锁定输入框，启动定时器
- **暂停/继续**: 切换定时器启停状态
- **结束**: 停止定时器，弹出统计报告，询问新桌数，重置引擎和界面

`on_simulation_step()`: 每 500ms 执行 `engine.step()` + `restaurant_view.update_view()`。

Qt 信号连接: `ControlPanel.preferences_changed` → `engine.update_preferences`。

### 5.2 `control_panel.py` — 操控面板

**ControlPanel** (QWidget 子类) 位于窗口右侧，包含:
- 4 个偏好输入卡片 (每种偏好一个 QSpinBox，范围 0-100)，各有不同背景色
- 一个 PreferencePieChart 饼图组件

`on_pref_changed()`: 任何 SpinBox 值变化时触发 → 实时更新饼图 → 若总和=100% 则发射 `preferences_changed` 信号，通知引擎更新偏好比例。

`set_inputs_enabled()`: 仿真运行时禁用所有输入框，防止中途改参。

### 5.3 `restaurant_view.py` — 餐厅视图

**RestaurantView** (QGraphicsView 子类) 使用 Qt 图形框架绘制餐厅。

`init_restaurant_layout(restaurant)`: 遍历所有桌子，根据 Table2(60×60) 或 Table4(100×60) 绘制矩形桌子 + 椭圆形椅子。按行排列，横向超出 600px 自动换行。椅子位置根据桌型计算坐标。

`update_view(restaurant)`: 每 tick 调用，遍历所有座位，根据 `table.seats[i]` 的值切换椅子颜色: 非0→金色(有人)，0→蓝色(空)。不做全量重绘，只更新颜色。

内部维护 `table_items` 字典作为 Model↔View 缓存桥接。

### 5.4 `charts.py` — 饼图

**PreferencePieChart** (QWidget 子类) 嵌入 matplotlib 饼图。通过 `FigureCanvasQTAgg` 将 matplotlib 的 Figure 包装为 Qt 控件。

`update_chart(sizes)`: 接收 4 个数值，绘制彩色饼图。颜色与 ControlPanel 一致 (蓝/粉红/绿/橙)。

### 5.5 `table_count_dialog.py` — 桌数弹窗

**TableCountDialog** (QDialog 子类) 模态弹窗，让用户设置双人桌和四人桌数量。

- 双人桌 0~50 张 (每张占 4 单位空间)
- 四人桌 0~33 张 (每张占 6 单位空间)
- 实时校验总空间 ≤200，超限禁用"确定"按钮

`get_table_counts()` 是 `@classmethod` 工厂方法，一行调用即可弹出窗口并返回结果。

## 七、核心数据流

```
[用户设置桌数] → TableCountDialog → SimulationEngine.__init__
[用户设置偏好] → ControlPanel.SpinBox → Signal → Engine.update_preferences

每 500ms 一个 Tick:
  QTimer.timeout
    → MainWindow.on_simulation_step
      → Engine.step()
        → _process_departures()   # 释放到期学生的座位
        → StudentGenerator.generate_batch(time, ratios)  # 倒U型人流
        → SeatAllocator.allocate(student)  # 两阶段匹配
        → SatisfactionCalculator.calculate(student) # 评分
      → RestaurantView.update_view(restaurant)  # 刷新椅子颜色
```

数据建模的核心是 `Table.seats` numpy 数组 — Allocator 写入、Engine 清空、View 读取渲染，三者围绕同一份数据协作。
